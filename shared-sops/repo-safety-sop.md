# Repo Safety SOP

## Purpose

This SOP protects active workspaces while agents inspect or modify documentation and code.

## Default Rules

- Check current directory and Git status before edits.
- Do not delete, move, rename, or overwrite files without approval.
- Do not modify active product code unless the task explicitly allows it.
- Do not read or expose secrets.
- Do not push or commit automatically.
- Keep workspace audits top-level unless deeper inspection is approved.

## Review Checklist

- Scope confirmed.
- Dirty worktree understood.
- Files to edit identified.
- Prohibited paths identified.
- Validation plan documented.
- Human approvals captured before risky actions.
