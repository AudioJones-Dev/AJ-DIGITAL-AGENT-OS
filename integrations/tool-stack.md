# Tool Stack

This file records the expected tools for the AJ Digital Agent OS ecosystem. Unconfirmed production integrations are marked as planned or optional.

| Tool | Status | Role |
| --- | --- | --- |
| Codex Desktop | Active | Local repo inspection, documentation edits, code changes, validation, Git-aware execution. |
| Claude CoWork | Active/Planned | Planning, synthesis, documentation, and control-room collaboration. |
| Claude Code | Planned | Repo-aware implementation assistant when explicitly assigned. |
| Hermes Agent | Planned | Orchestration, memory, routing, and policy enforcement layer. |
| OpenClaw | Planned/Optional | Future execution or routing component. |
| Obsidian | Active/Planned | Knowledge base, briefs, decisions, and operator context. |
| GitHub | Active | Source control, repo hosting, issues, pull requests, and release records. |
| Cloudflare | Planned/Active | DNS, domain control plane, wildcard routing, edge controls. |
| Vercel | Planned/Active | Web app hosting, preview deployments, production deployments. |
| Neon | Planned/Optional | Serverless Postgres for product and agent data. |
| Cloudflare R2 | Planned/Optional | Object storage for files, media, exports, and artifacts. |
| Upstash | Planned/Optional | Redis, queues, rate limits, and lightweight workflow state. |
| PostHog | Planned/Optional | Product analytics, event capture, and usage visibility. |

## Rule

Integrations are not production-ready just because they are listed here. Each integration needs setup verification, secret placement, ownership, and approval gates before production use.
